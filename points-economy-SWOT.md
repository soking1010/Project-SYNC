# 台灣點數經濟流通困境 × Project SYNC SWOT 分析

## 健康行為積點如何在「點數疲勞」時代找到差異化生存空間

---

> **摘要**：台灣點數經濟在 2026 年已進入「碎片化飽和」階段，消費者面臨點數疲勞、匯率損失與圍牆花園三大困境。本報告以 Project SYNC 的 hEXP/hMile 雙幣健康經濟系統為分析對象，從台灣點數市場現況切入，進行 SWOT 分析，並提出策略建議。

---

## 一、市場脈絡：台灣點數經濟的結構性問題

### 1.1 現況數據

| 指標 | 數據 |

| 消費者平均持有點數 App | 10+ 個 |

### 1.2 三大困境與 SYNC 的結構性迴避

| 困境 | 傳統點數痛點 | SYNC 的結構性迴避 |
|------|------------|-----------------|
| **匯率損失** | A 點數轉 B 點數縮水 20-40% | hEXP 不可轉讓（Soulbound），從根本消除「兌換匯率」問題 |

**關鍵洞察**：SYNC 的雙幣設計（hEXP 靈魂綁定 + hMile 流通）恰好跳脫了傳統「單一積點」系統的三大困境，但也帶來了全新的挑戰。

### 2.2 Strengths（優勢）

#### S1：雙幣分離設計 — 從根本迴避「匯率陷阱」

台灣點數經濟最大的痛點是「點數兌點數」的匯率損失。SYNC 的 hEXP/hMile 雙幣設計**從架構層面消除了這個問題**：

- **hEXP（靈魂綁定）**：不可轉讓、不可交易，不存在「匯率」的概念
- **hMile（流通層）**：單向從 hEXP 轉換而來（100:1），有嚴格的月度上限（50 hMile/月）

這種設計讓 SYNC 不需要進入「跨品牌點數互轉」的紅海戰場，也避免了傳統點數的「通膨貶值」問題。

**與傳統點數的結構性差異**：
```
傳統點數：
  賺取 → 累積 → 兌換（匯率損失）→ 過期

SYNC 雙幣：
  行為 → hEXP 累積（不可轉讓）→ 達標轉換 hMile → 兌換
          ↑                              ↑
       衰減+保底                    月度上限+行為門檻
```

#### S2：行為驗證壁壘 — 點數的「來源可信度」

台灣現行點數的核心問題之一是「濫發」——消費就送點、註冊就送點，點數的含金量極低。

SYNC 的 hEXP 每一筆都對應**可驗證的健康行為事件**（穿戴裝置 API、QR/NFC 簽到、健保系統對接、AI 驗證）。這讓 hMile 成為市場上唯一一種「經過行為驗證的注意力憑證」。

**對活水夥伴（商家）的價值主張**：
> 「你投放的每一塊獎勵，觸及的都是真實行為過的用戶，不是機器人、不是羊毛黨。」

#### S3：P4P 制度對接 — 「國家級點數系統」的延伸

台灣健保 P4P 本質上就是一個**醫療版的論質計酬積點系統**——醫療團隊透過品質指標達標來獲得額外獎勵金。SYNC 的 hEXP 可以被理解為**P4P 的病患端延伸**：

```
P4P 現行體系：
  醫療團隊 → 達成品質指標 → 獲得 P4P 獎勵金

SYNC 延伸層：
  病人 → 完成行為任務 → 累積 hEXP → 轉換 hMile → 兌換獎勵
                                ↑
                    同時協助醫療團隊達成 P4P 指標
                    （回診率、HbA1c 達標率提升）
```

這讓 SYNC 不只是「又一個點數系統」，而是**鑲嵌在國家醫療制度裡的行為激勵層**。

#### S4：非零和經濟模型 — 「行為價差」而非「匯率價差」

傳統點數經濟是零和的：消費者獲得的點數價值，等於企業讓渡的利潤。當多個品牌在同一個消費者身上競爭時，就產生了「點數戰爭」。

SYNC 的活水夥伴模式創造了**正和循環**：
- 活水夥伴投入任務與獎勵 → 獲得「行為驗證的精準受眾」
- 病人完成行為任務 → 獲得健康改善 + hEXP/hMile
- 醫療端 P4P 指標達標 → 獲得獎勵金
- 社會面減少醫療支出 → 全民受益

**三方共贏結構**：每一筆 hEXP 的產生，同時服務了病人（行為改善）、醫療（指標達標）、商家（精準導流）三方利益。

#### S5：衰減+保底設計 — 解決「點數過期焦慮」

台灣每年 20-30% 的點數因過期而被回收，造成消費者的「過期焦慮」和「點數疲勞」。

SYNC 的衰減機制更加人性化：
- **不是硬性過期**：而是逐步衰減，鼓勵回歸
- **20% 保底**：承認過去的努力，離開再回來不會從零開始
- **Streak Multiplier**：連續參與有加成，但不懲罰中斷

這種設計比「到期作廢」更符合慢性病照護的長期性質——病人可能因住院、季節、生活變故而暫時中斷，系統不應因此「懲罰」他們。

### 2.3 Weaknesses（劣勢）

#### W1：流通場景稀缺 — hMile「能換什麼」？

**這是 SYNC 最大的結構性劣勢。**

LINE Points 之所以是「硬通貨」，是因為它在便利商店、超市、線上購物、LINE Pay 結帳等無數場景都能用。SYNC 的 hMile 在 Phase 1 幾乎沒有流通場景：

| 比較 | LINE Points | SYNC hMile |
|------|-------------|-----------|
| 可使用通路數 | 10,000+ | Phase 1: 3-5 個合作商家 |
| 兌換直覺性 | 結帳時直接扣抵 | 需要額外操作 |
| 消費者認知 | 「LINE 點數 = 幾乎等於現金」 | 「hMile 是什麼？」 |

**風險**：如果病人累積了 hMile 卻發現「沒什麼好換的」，將嚴重打擊參與動機。

#### W2：認知門檻極高

台灣消費者已經被「消費送點數」訓練了 20 年。SYNC 要求他們理解：
1. hEXP 和 hMile 是兩種不同的東西
2. hEXP 不能直接換東西
3. 需要持續行為才能轉換為 hMile
4. hMile 有月度上限
5. 不參與的話 hEXP 會衰減

**對目標族群（60 歲以上慢性病患）而言**，這個認知負擔遠超傳統「消費送點」模型。

#### W3：冷啟動的雞蛋問題

```
沒有用戶 → 活水夥伴不願進場（沒有流量）
            ↑                    │
            │                    ↓
            └── 沒有獎勵場景 ← 用戶沒有動力
```

傳統點數系統透過「消費行為天然存在」來解決冷啟動——人總是要去便利商店買東西。但 SYNC 的「健康行為」需要額外的動力啟動，缺乏天然的行為觸發點。

#### W4：hMile 匯率定價的困難

hMile 的價值取決於「活水夥伴願意用多少實質獎勵來兌換」。但在早期：
- 合作商家少 → 定價缺乏市場依據
- 用戶量少 → 商家認為「精準受眾」的價值有限
- 與 LINE Points 比較 → hMile 的「感知價值」天然偏低

**定價兩難**：定太低，用戶覺得不值得；定太高，商家覺得太貴。

#### W5：技術複雜度高

雙幣系統、衰減機制、行為驗證、P4P 串接、TW DIW VC 化⋯⋯每一層都是獨立的技術挑戰。Phase 1 的簡化版（hEXP + 簡單積點）雖降低了複雜度，但完整實現仍需大量開發資源。

### 2.4 Opportunities（機會）

#### O1：點數疲勞的「反向需求」

消費者對傳統「消費送點」模式已經產生疲勞。SYNC 的「行為送點」模式可能正好切中「我想做一些有意義的事來獲得獎勵」的新興心理需求。

**潛在定位**：
> 「不是因為你花了多少錢，而是因為你為自己的健康做了多少。」

在 ESG 意識抬頭的 2026 年，「有意義的點數」可能比「又多一個折扣」更有吸引力。

#### O2：888 計畫 —— 健保制度性窗口

888 計畫目標將 80% 三高患者（超過 400 萬人）納入照護網，且明確納入**生活型態諮詢**為品質指標。這等於國家層級在為「行為介入」開路：

- **政策紅利**：SYNC 的 AI 健康對話可直接計為「生活型態諮詢」
- **覆蓋範圍**：400 萬人的潛在觸及量
- **給付支持**：P4P 獎勵金為 SYNC 提供了非使用者付費的收入來源

#### O3：家醫大平台 API 串接

2025 年底上線的家醫大平台將整合個人健康資料 + AI 風險分級。SYNC 若能成為其「行為數據供應商」：

- 不需要自建龐大的流通場景
- 行為數據本身就是「產品」
- 可借平台的覆蓋量快速規模化

#### O4：點數代幣化趨勢

2026 年部分業者開始嘗試將點數區塊鏈化。SYNC 的 hMile 在 Phase 3 本就規劃鏈上化，這讓 SYNC 有機會**領先傳統點數系統一步**進入代幣化時代：

- 透明匯率結算（智能合約）
- 跨品牌即時清算
- 選擇性揭露（TW DIW VC）保護隱私

#### O5：ESG 與影響力投資熱潮

SYNC 的「健康行為點數」天然具備 ESG（Social）與 SDG（Good Health and Well-being）屬性。這讓 SYNC 在募資時具備：

- Google.org Impact Challenge 的契合度
- 國發基金社會創新組的申請資格
- 企業 CSR/ESG 報告的量化影響力指標

### 2.5 Threats（威脅）

#### T1：LINE Points 的「硬通貨壓制效應」

LINE Points 是台灣流通性最高的點數，對消費者而言幾乎等於現金。任何新點數系統都必須面對「為什麼不直接給我 LINE Points？」的質疑。

**具體威脅**：
- 活水夥伴可能直接說「我用 LINE Points 獎勵病人就好」
- 使用者可能只關心「hMile 能不能換成 LINE Points」
- 競品（智抗糖等）可能直接採用 LINE Points 獎勵

#### T2：既有數位照護競品

健保署已選定 4 家數位照護廠商試辦（智抗糖、WaCare、智齡科技、遠傳電信）。他們已有：
- 健保署官方關係
- 既有用戶基礎
- 成熟的產品
- 可能搶先佔位 888 計畫

#### T3：點數法規風險

hMile 在 Phase 1-2 定位為「平台內點數」，但若 Phase 3 鏈上化：
- 可能觸發虛擬通貨相關法規
- 金管會對「代幣化點數」的監管態度尚不明朗
- 若被歸類為「有價證券」將面臨嚴格規範

#### T4：用戶習慣慣性

目標族群（60 歲以上慢性病患）：
- 對新型態「點數」概念接受度低
- 已習慣「看診 → 拿藥 → 回家」的線性流程
- 數位素養參差不齊
- LINE 使用以通訊為主，不熟悉 Bot 互動

#### T5：醫療端採用阻力

P4P 參與醫師已經面臨繁重的文書工作：
- 多一個系統 = 多一份負擔
- 即使 SYNC 定位為「減輕負擔」，初期學習成本仍存在
- 基層診所 P4P 參與率僅 8%，觸及面有限

## 四、策略優先序建議

### 第一優先：Phase 0-1 的「去點數化」包裝

**核心洞察**：SYNC 在台灣點數市場的最佳策略不是「成為又一個點數系統」，而是**完全避開「點數」的框架**。

|---------|---------|----------|
| 病人 | 「做完健康任務就有小禮物」 | 「累積 hEXP 轉換 hMile」 |

| 投資人 | 「健保 P4P 延伸的行為介入平台」 | 「我們要做健康版的 LINE Points」 |

### 第三優先：與既有點數生態的「寄生策略」

不要試圖取代 LINE Points 或 OpenPoint，而是**寄生在他們的流通網絡上**：

1. **LINE Points 作為兌換出口**：hMile 可以小額兌換 LINE Points（1 hMile = 5-10 LINE Points），利用 LINE 的流通基礎設施
2. **OpenPoint 合作診所聯名**：與統一超商合作「健康打卡送 OPEN 小將徽章」，借用既有品牌認知
3. **全聯福利點健康購物**：hMile 兌換健康食品專屬折扣券，不直接競爭點數

## 五、結論

### Project SYNC 在台灣點數經濟中的獨特定位

```
台灣點數經濟光譜：

  消費驅動 ◄──────────────────────────────► 行為驅動
  │                                          │
  OpenPoint   LINE Points   健保存摺   SYNC hEXP/hMile
  FamiPoint   蝦幣/mo幣                      │
  銀行紅利    Happy Go                        │
  │                                          │
  流通性最強                            行為含金量最高
  行為含金量最低                        流通性最弱（目前）
```

**SYNC 的戰略位置**：光譜最右端——「行為含金量最高，流通性最弱」。

**核心策略**：不要向左移（提升流通性的代價是稀釋行為含金量），而是**讓光譜右端的價值被看見**：

1. 透過 P4P 制度對接，讓 hEXP 的「行為含金量」直接轉化為醫療品質指標的改善
2. 透過家醫大平台串接，讓行為數據本身成為比「可兌換金額」更有價值的產品
3. 透過 ESG/影響力投資敘事，讓「行為驗證」的社會價值獲得資本市場認可

**一句話總結**：

> **SYNC 不是要在點數紅海裡競爭流通性，而是要在健康行為這個藍海裡，定義一種全新的「價值含金量」標準。**
