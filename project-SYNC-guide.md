# 【專案建議書】AI 驅動全人健康守護計畫

## 預防醫學的行為設計與社會處方箋實踐

**提案人：** 黃仁穩 主治醫師／Soking UX 顧問團隊

**關鍵對標：** Google.org $30M Impact Challenge & 衛福部「全球首創 AI 健康網」

## 產品哲學

- **預防醫學出發** — 以健康促進取代疾病治療，將介入時間點前移至行為階段
- **政府開放資料** — 善用健保署「健康存摺」等公共數據基礎設施
- **民間企業參與** — 引入科技業與設計業能量，加速醫療場域數位轉型
- **個人資料可攜性** — 確保病患對自身健康數據擁有主權與自主授權能力

## 一、政策與市場背景（Why Now?）

### 國家政策對接

衛福部於 2026/03/04 宣布與 Google 合作導入 Gemini 衛教助理至「健康存摺」。本計畫旨在補足「診間外行為落實」的最後一哩路。

### 國際資金槓桿

Google.org 目前提供全球 3,000 萬美元資助，專注於 Agentic AI（代理式 AI）於公共服務的應用。申請截止日為 **2026/04/03**。

---

## 二、核心問題（The Gap）

目前慢性病管理（DM / HTN）面臨兩大瓶頸：

1. **醫囑落實率低：** 患者離開診間後，生活型態改變動機難以持續。
2. **社交孤立惡化：** 高齡患者缺乏社會連結，導致心理韌性下降，增加非必要醫療支出。

---

## 三、解決方案：AI 數位行為基礎建設（The Solution）

本計畫將發展一套結合 **UX 行為設計** 與 **代理式 AI** 的數位基礎設施：

### 數位藥方（Digital Prescription）

醫師於診間開立「行為處方箋」，AI 代理人根據病人實時生理數據（穿戴裝置）與生活情境，主動觸發遊戲化微任務。

### 自動化社會處方箋

系統自動串接社區資源（如銀髮據點、藝術治療），由 AI 擔任媒合代理人，將醫療照護延伸至社會參與，實踐全人醫療。

---

## 四、執行亮點與創新性（Innovation）

### 從被動到主動（Agentic AI）

不同於傳統 App 記錄，本系統能主動感知病人行為窗口並進行心理干預。

### 醫社數據閉環

將病人的社會參與度、生活習慣回傳至醫師後台，作為診間診斷與「健保論質計酬」的客觀依據。

## 相關資源

### 1. 健康存摺 SDK（My Health Bank SDK）

這是將病患在健保署的醫療數據匯入 App 的正式管道。

- **申請作業指南（PDF）：** 健康存摺 SDK 申請作業說明
- **機關登記入口：** 健保資訊服務系統（Internet）
  - 路徑：登入後至「其他服務」>「健康存摺 SDK 機關登記作業」
- **常見問題與應用計畫書下載：** SDK 健康服務連結專區

### 2. 大家醫平台與健保資訊申報

這是對接「888 計畫」與 P4P 數據回傳的核心平台。

- **健保資訊服務系統（VPN / Internet）：** med.nhi.gov.tw
  - 這是診所端執行醫事機構登記、授權人員管理及未來數據拋轉的主要入口。
- **電子病歷平台介接說明（衛福部）：** 民眾線上申辦電子病歷服務平台介接說明
  - 提供技術規格如 API 介接、SFTP 傳輸與加密規範。

### 3. 行動應用資安檢測（MAS 標章）

醫療 App 介接健保數據前，必須通過的資安驗證門檻（建議 L2 等級）。

- **行動應用資安聯盟官網：** www.mas.org.tw
- **檢測實驗室名錄：** 合格檢測實驗室查詢
  - 可在此尋找具備 TAF 認證、可發放 MAS 標章的檢測機構（如台灣電子檢驗中心等）。

### 4. 其他輔助資源

- **醫事憑證管理中心（HCA）：** hca.nat.gov.tw
  - 若涉及診所端的醫事機構卡申辦或授權問題，需透過此網站辦理。
- **工商憑證管理中心（MOEACA）：** moeaca.nat.gov.tw
  - 非醫事機構（如開發公司）申請 SDK 時，需以此憑證進行身分識別。
