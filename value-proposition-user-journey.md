# Project SYNC — 價值主張與用戶旅程分析

## 破壞式創新視角：JTBD × 峰值體驗 MOT × 增長飛輪

---

> **核心觀點**：Project SYNC 的終端使用者不是「病人」——他們是「被困在醫療系統裡，想要奪回日常生活主控權的人」。每一個慢性病患者心中都有一個 Job to Be Done：**「讓我覺得自己還有能力管好自己的身體。」** 切中這個 Job，就能打開一整條從引流到推薦的增長通道。

---

## 一、終端使用者分類：不是依「疾病」，而是依「行為動力結構」

### 1.1 為什麼不能用「糖尿病患者」「高血壓患者」來分類？

傳統醫療系統按疾病分類病人。但從 JTBD 的角度，**同一種疾病的不同病人，其行為動力結構可能完全不同**。一個剛被確診的 55 歲上班族和一個已經共病 10 年的 72 歲獨居老人，他們的 Job 截然不同，即使醫師開出的處方箋幾乎一樣。

### 1.2 四種行為動力原型（Behavioral Archetype）

```
                高自我效能感
                    │
           ┌────────┼────────┐
           │        │        │
           │   A 型  │   B 型  │
           │ 奮鬥者  │ 探索者  │
           │        │        │
  低社會連結 ────────┼──────── 高社會連結
           │        │        │
           │   C 型  │   D 型  │
           │ 漂流者  │ 從眾者  │
           │        │        │
           └────────┼────────┘
                    │
                低自我效能感
```

#### 🅰 奮鬥者（The Striver）— 「我知道該做什麼，但環境不支持我」

**人物原型**：王大姐，58 歲，二型糖尿病確診 2 年

```
背景：
- 中小企業行政主管，工時長、壓力大
- 確診後認真研究飲食控制，買了血糖機
- 問題不是「不知道」，而是「做不到」——加班到 8 點還要自己煮健康餐？
- 老公和小孩不配合飲食調整，覺得她「太誇張」
- 每次回診 HbA1c 沒降，覺得「都是我的錯」
```

**Jobs to Be Done**：

| 層次 | Job 描述 |
|------|---------|
| **功能性** | 在忙碌的工作日中，找到可執行的最小健康行為 |
| **情感性** | 證明「我有在努力」——不只是對醫師，更是對自己 |
| **社會性** | 讓家人理解這不是「她一個人的事」 |

**關鍵洞察**：奮鬥者不需要「更多資訊」，她需要的是**「環境的共犯」**——一個幫她在現實限制中找到可行路徑的系統。

**SYNC 的 Job 回應**：
> 「不是叫你做一個完美的糖尿病患者，而是幫你在今天的行程裡，找到一個 15 分鐘的健康縫隙。」

---

#### 🅱 探索者（The Explorer）— 「我不只想管好血糖，我想活得更好」

**人物原型**：李大哥，65 歲，退休教師，高血壓 + 初期 CKD

```
背景：
- 退休後時間充裕，積極參與社區活動
- 加入了糖尿病病友群組，常分享健康資訊
- 主動研究營養學，每天量血壓並記錄在 Excel
- 問題不是「缺乏動力」，而是「找不到系統化的進步框架」
- 想知道自己做的事到底有沒有效
```

**Jobs to Be Done**：

| 層次 | Job 描述 |
|------|---------|
| **功能性** | 將散亂的健康行為（走路、飲食、社交）整合為可追蹤的進度系統 |
| **情感性** | 感受到「越做越好」的持續進步感——遊戲化的升級體驗 |
| **社會性** | 成為社區中的「健康達人」，幫助其他人 |

**關鍵洞察**：探索者是 SYNC 的**超級用戶候選人**。他們自帶動力，需要的是一個「讓努力被看見」的舞台。

**SYNC 的 Job 回應**：
> 「你已經在做的每一件好事，終於有一個系統幫你記錄、量化、展示。」

---

#### 🅲 漂流者（The Drifter）— 「反正好不了了，差不多就好」

**人物原型**：陳伯伯，72 歲，獨居，糖尿病 + 高血壓共病 15 年

```
背景：
- 老伴過世 3 年，兒女在北部
- 每天的行程：起床 → 看電視 → 吃便當 → 看電視 → 睡覺
- 會按時吃藥（因為藥盒是固定的），但不運動、不控制飲食
- 對健康的態度：「活到這把年紀，差不多了」
- 唯一的社交：隔壁鄰居偶爾來串門
- 用 LINE 看兒子傳的照片，不太主動發訊息
```

**Jobs to Be Done**：

| 層次 | Job 描述 |
|------|---------|
| **功能性** | （他不覺得自己有需要做的事） |
| **情感性** | 不想成為家人的「負擔」 |
| **社會性** | 有人關心自己——但不想被「管」 |

**關鍵洞察**：漂流者的真正 Job 不是「改善健康」，而是**「不想讓兒女擔心到把我送進安養院」**。這個 Job 的力量比任何衛教都強。

**SYNC 的 Job 回應**：
> 「不是要你改變生活方式，而是讓你的兒女看到——爸爸今天有出門走路了。」

---

#### 🅳 從眾者（The Follower）— 「大家都在做，那我也做」

**人物原型**：張阿姨，68 歲，社區大學學員，高血脂

```
背景：
- 社區大學的常客，上書法課、太極拳課
- 健康問題不嚴重，但醫師提醒要注意
- 對新東西的態度：「隔壁的林太太在用，那我也試試」
- 使用 LINE 非常活躍，加了十幾個群組
- 最大動力：和朋友一起做什麼事
```

**Jobs to Be Done**：

| 層次 | Job 描述 |
|------|---------|
| **功能性** | 和朋友一起做健康相關的活動 |
| **情感性** | 歸屬感——「我們這群人在一起做一件有意義的事」 |
| **社會性** | 有聊天的話題——「你今天集了幾分？」 |

**關鍵洞察**：從眾者是 SYNC 的**病毒式傳播引擎**。她不會因為自己的健康需求加入，但會因為「朋友都在用」而加入，然後拉更多人進來。

**SYNC 的 Job 回應**：
> 「你和林太太本週都在走路挑戰裡，她走了 12,000 步，你呢？」

### 1.3 四種原型的分布與策略意義

```
用戶量分佈（估計）：

  漂流者 C ████████████████████ 40%   ← 最大群體，但最難啟動
  從眾者 D ███████████████ 30%         ← 最佳傳播引擎
  奮鬥者 A ████████ 15%               ← 最高 LTV，但規模有限
  探索者 B ██████ 10%                  ← 超級用戶，內容生產者
  其他     ██ 5%

Phase 1 切入策略：B → D → A → C（不是從最大群體開始）
```

**違反直覺的策略選擇**：不要從最大的群體（漂流者）開始。從探索者（B）開始，因為他們：
1. 自帶動力，不需要推
2. 會產生內容和社群動態
3. 會成為從眾者（D）的引爆點
4. 他們的成功故事能吸引奮鬥者（A）
5. 漂流者（C）最後才透過家屬和社區據點被「帶入」

## 二、JTBD 深度解構：醫囑落實的「競爭性消費」

### 2.1 真正的競品不是其他 App——是「沙發」

Clayton Christensen 的 JTBD 理論強調：**產品的真正競爭者不是同類產品，而是消費者目前用來解決同一個 Job 的替代方案。**

對慢性病患者而言：

| Job | 目前的「雇用方案」（Hiring） | SYNC 要取代的是什麼？ |
|-----|---------------------------|---------------------|
| 「管好血糖」 | 吃藥（自動化，不需動腦） | 不需要取代，而是在吃藥之外加一層 |
| 「每天運動」 | 看電視取代了運動（因為運動是痛苦的） | 把「運動」重新定義為「出門走 15 分鐘」 |
| 「控制飲食」 | 「差不多就好」（因為改變飲食太複雜） | 不要求改變，而是「記錄你吃了什麼」|
| 「不想麻煩家人」 | 不告訴家人（迴避策略） | 讓「做了健康行為」成為家人安心的信號 |
| 「打發時間」 | LINE 群組聊天、看 YouTube | 讓「完成健康任務」成為新的社交貨幣 |

**破壞式創新切入點**：

SYNC 的競爭對手不是智抗糖、不是 Fora Health、不是任何健康 App。SYNC 的競爭對手是：
- 沙發上的電視遙控器
- 便利商店的便當
- 「算了，差不多就好」的心理慣性

**這意味著 SYNC 不能用「更好的功能」來競爭，而要用「更低的啟動門檻」來競爭。**

### 2.2 非消費市場（Non-Consumption）的機會

目前有 80% 的慢性病患者「不落實醫囑」——這不是因為市場上沒有好的健康管理 App，而是因為**他們根本不在「健康管理」這個市場裡消費**。

```
                     高端市場
                        │
  ┌──────────────────────────────────────────┐
  │  Apple Health + 穿戴裝置 + 專業營養師     │ ← 探索者在這裡
  │  智抗糖 + 血糖機 + App 記錄               │ ← 奮鬥者勉強在這裡
  ├──────────────────────────────────────────┤
  │                                          │
  │           非消費市場                       │ ← 80% 的人在這裡
  │           「不使用任何工具」               │
  │           「吃藥就好」                     │
  │           「醫師講完就忘了」               │
  │                                          │
  │       ← Project SYNC 切入這裡 →          │
  │       用 LINE Bot + 極簡任務              │
  │       服務「目前什麼都不做的人」            │
  │                                          │
  └──────────────────────────────────────────┘
                        │
                     低端市場
```

**SYNC 的破壞式定位**：不是做一個「更好的健康管理 App」，而是做一個「讓不用任何工具的人開始用第一個工具」的入口。

## 三、峰值體驗 MOT（Moments of Truth）設計

### 3.1 MOT 理論在 SYNC 的應用

丹尼爾·乘坦的《體驗經濟》與齊普·希斯的《關鍵時刻》指出：人不會記住一段體驗的平均感受，而是記住**峰值（Peak）**、**谷底（Valley）**和**結尾（End）**。

SYNC 的用戶旅程中，需要刻意設計以下 MOT：

### 3.2 引流階段 MOT — 「破冰的 30 秒」

#### MOT-1：「原來這麼簡單」時刻（Zero Friction Entry）

**場景**：陳伯伯在診間，黃醫師說「我幫你加一個 LINE 好友」

```
                     傳統路徑（死亡漏斗）
                     ────────────────────
  「下載 App」 → 「註冊帳號」 → 「填寫基本資料」 → 「設定目標」
       ↓              ↓              ↓              ↓
    放棄 30%       放棄 25%       放棄 20%       放棄 15%
                                            最終轉換率：~25%

                     SYNC 路徑（零摩擦入口）
                     ────────────────────
  「掃這個 QR Code」 → 「加好友」 → 「收到第一則訊息」
       ↓                  ↓              ↓
    放棄 5%            放棄 3%         已經在用了
                                            最終轉換率：~92%
```

**峰值設計**：
- 醫師親手拿出 QR Code（權威感 + 個人化 → 峰值情緒）
- 加好友後 **3 秒內** 收到歡迎訊息（即時回饋 → 確認「成功了」）
- 歡迎訊息不問任何問題，只說：「醫師特別為你開了健康任務，明天早上 8 點我提醒你 😊」

**為什麼這是 Peak**：病人在診間的信任度最高——醫師剛面對面叮囑了，情緒上處於「我要好好做」的高點。SYNC 必須在**這個高點的 30 秒內完成轉換**，否則出了診間就會迅速衰減。

#### MOT-2：「被看見」時刻（First Recognition）

**場景**：陳伯伯加好友的隔天早上 8:00

```
LINE 推播：
┌──────────────────────────────────────────┐
│                                          │
│  早安，陳伯伯！                            │
│                                          │
│  黃醫師說您昨天血壓量得不錯 👍              │
│  今天的小任務：                            │
│                                          │
│  🚶 飯後散步 15 分鐘                      │
│                                          │
│  完成後跟我說一聲就好                      │
│                                          │
│  ┌────────┐                              │
│  │ 我走完了 │                              │
│  └────────┘                              │
│                                          │
└──────────────────────────────────────────┘
```

**峰值設計**：
- **叫名字**（「陳伯伯」不是「用戶」）
- **提到醫師**（延續診間的信任連結）
- **只有一件事**（認知負荷最低）
- **完成按鈕只有一個**（不需要打字）

**為什麼這是 Peak**：從「加好友」到「第一次互動」是存活率的關鍵斷層。如果第一則訊息讓人覺得「被關心了」而不是「被推銷了」，留存率可以從 40% 跳到 70%。

### 3.3 轉換階段 MOT — 「啊，真的有用」

#### MOT-3：「三天連續」成就解鎖（Streak Celebration）

**場景**：陳伯伯連續三天回報「走完了」

```
LINE 推播（第三天完成後）：
┌──────────────────────────────────────────┐
│                                          │
│  🎉 連續 3 天達標！                        │
│                                          │
│  陳伯伯，你做到了！                        │
│  這是本社區裡 87% 的人都做不到的事。       │
│                                          │
│  ┌─────────────────────────────┐         │
│  │  📊 你的本週成績              │         │
│  │                              │         │
│  │  步行    ███░░  3/5 天       │         │
│  │  hEXP    ⭐ 35 分            │         │
│  │  連續    🔥 3 天             │         │
│  │                              │         │
│  │  再 2 天就解鎖「銅牌行動者」！│         │
│  └─────────────────────────────┘         │
│                                          │
│  要不要邀請家人看你的成績？               │
│  ┌─────────┐ ┌──────────┐               │
│  │ 分享給兒子 │ │ 先不用了  │               │
│  └─────────┘ └──────────┘               │
│                                          │
└──────────────────────────────────────────┘
```

**峰值設計**：
- **「87% 的人做不到」**（社會比較 → 自豪感 → Peak Emotion）
- **視覺化進度條**（看見進步 → 正向回饋循環）
- **「再 2 天就解鎖」**（接近完成效應 → 動力延續）
- **分享機制植入**（但不強迫 → 自然推薦入口）

**為什麼第三天是 Peak**：行為心理學研究顯示，連續 3 天是「習慣萌芽」的第一個轉折點。在第三天給予強烈的正向回饋，能讓「試試看」轉變為「我要繼續」。

#### MOT-4：「醫師看到了」回診體驗（Doctor Acknowledgment）

**場景**：陳伯伯 3 個月後回診，黃醫師打開儀表板

```
黃醫師（看著螢幕）：
「陳伯伯，我看到你這三個月走了 62 天，
  社區活動去了 8 次，太棒了！」

「你看，你的 HbA1c 從 8.2 降到 7.6 了，
  這跟你的努力一定有關係。」

陳伯伯心裡：
  「醫師看到了！我的努力不是白費的！」
```

**為什麼這是整個旅程最重要的 Peak**：

這是**SYNC 的「魔法時刻」**——病人第一次體驗到「我的日常行為」和「醫師看到的數字」之間有直接關聯。這個時刻具有三重力量：

1. **行為 → 數字的因果確認**：打破「反正做不做都一樣」的習得性無助
2. **醫師的權威認可**：比任何 App 的徽章都有力量一萬倍
3. **下一個 3 個月的動力注入**：「既然有用，那我繼續做」

**設計原則**：SYNC 的醫師儀表板不是給醫師看「數據」，是給醫師一個「讚美病人的素材」。

### 3.4 重複階段 MOT — 「停不下來」

#### MOT-5：「我的鄰居也在」社群引爆（Community Ignition）

**場景**：從眾者張阿姨看到 LINE 群組的訊息

```
社區 LINE 群組：
┌──────────────────────────────────────────┐
│  林太太：大家今天走了多少步？                │
│  我剛看 SYNC 說我這週走了 28,000 步！       │
│  🏆 社區排行第三名！                        │
│                                          │
│  陳伯伯：哈哈我第五名 差你好多               │
│                                          │
│  王大姐：@張阿姨 妳怎麼還沒加入              │
│  掃這個就可以了 [QR Code]                  │
│                                          │
│  張阿姨：好啦好啦 明天幫我弄 🤣              │
└──────────────────────────────────────────┘
```

**峰值設計**：
- **社區排行榜**不是「全國排名」，而是「你家附近 50 人」的圈子
- **用真名而非匿名**（社區裡大家本來就認識）
- **比的是「有沒有做」而非「做多好」**（避免打擊低能力者）

**為什麼這是重複使用的 Peak**：

當健康行為從「我一個人的事」變成「我們社區在一起做的事」，**dropout 率會從 60% 降到 20%**。因為退出的心理成本從「我對不起自己」升級為「明天在社區遇到林太太會不好意思」。

#### MOT-6：「我換到東西了！」首次兌換（First Redemption）

**場景**：陳伯伯累積了 30 個兌換積點

```
LINE 推播：
┌──────────────────────────────────────────┐
│                                          │
│  🎁 陳伯伯，你的健康積分可以兌換了！       │
│                                          │
│  你累積了 30 積分，可以兌換：              │
│                                          │
│  ☕ 社區早餐店 — 豆漿一杯（20 積分）       │
│  🥬 健康蔬果箱 — 9 折券（25 積分）         │
│  💊 合作藥局 — 保健品折扣（30 積分）        │
│                                          │
│  ┌────────────┐                          │
│  │ 我要換豆漿！ │                          │
│  └────────────┘                          │
│                                          │
└──────────────────────────────────────────┘
```

**峰值設計**：
- **第一次兌換必須「驚喜地快」**——不能讓人累積 3 個月才能換東西
- **兌換物是「日常的小確幸」**而非「大獎」——豆漿 > 健身房月費
- **兌換地點是「走路就到的」**——社區早餐店 > 百貨公司

**為什麼這是 Peak**：首次兌換是「hMile 有價值」的具體證明。如果這個體驗太慢或太麻煩，用戶會認定「積分沒用」。**首次兌換要在加入後 2 週內發生。**

**為什麼這是推薦的 Peak**：

漂流者（C 型）的真正 Job 是「不想麻煩家人」。當兒子看到爸爸的健康報告，心理發生的轉變是：

> 「原來爸爸有在動，我不用那麼擔心了。」→「這個東西真好用，我要推薦給同事的爸媽。」

**推薦不是從病人發出的——是從家屬發出的。** 家屬是最強的推薦引擎，因為：
1. 他們有強烈的情感動機（擔心父母）
2. 他們的社交圈充滿「同樣擔心父母」的人
3. 他們是數位原住民，轉發和分享毫無障礙

## 六、關鍵策略建議

### 6.1 Phase 0-1 必須驗證的三個假設

| # | 假設 | 驗證方法 | 最低可行數據 |
|---|------|---------|-----------|
| H1 | 醫師推薦是最有效的引流管道 | 追蹤各管道的轉化率 | 醫師推薦 > 自然搜尋 2 倍以上 |
| H2 | 第 3 天 Streak 是留存的轉折點 | Day 3 vs Day 7 活躍率比較 | Day 3 活躍者的 Day 30 留存 > 50% |
| H3 | 家屬報告是漂流者留存的關鍵 | 有/無家屬連結的留存率比較 | 有家屬連結者留存 > 無家屬者 2 倍 |

### 6.2 「去病人化」的敘事策略

**不要稱使用者為「病人」或「患者」。**

| 場景 | 不要說 | 應該說 |
|------|-------|-------|
| 引流 | 「管理你的糖尿病」 | 「每天做一件讓自己開心的小事」 |
| 任務 | 「控制血糖的運動處方」 | 「今天的 15 分鐘散步挑戰」 |
| 成就 | 「病情控制良好」 | 「你已經連續 30 天行動了！」 |
| 推薦 | 「推薦其他病友使用」 | 「邀請鄰居一起走」 |

**原理**：沒有人想被提醒自己是「病人」。SYNC 的語言體系應該讓人覺得自己是「行動者」——一個選擇為自己做些好事的人。

### 6.3 MOT 優先開發排序

| 優先級 | MOT | 原因 | 開發成本 |
|--------|-----|------|---------|
| P0 | MOT-1 零摩擦入口 | 沒有入口，一切歸零 | 低（LINE Bot 加好友） |
| P0 | MOT-2 第一則個人化訊息 | Day 1 留存決定一切 | 低（模板 + 姓名變數） |
| P0 | MOT-3 三天連續成就 | 習慣萌芽的轉折點 | 低（計數器 + Flex Message） |
| P1 | MOT-7 家屬週報 | 漂流者留存的唯一引擎 | 中（需要家屬連結功能） |
| P1 | MOT-4 回診體驗 | 整個旅程的最大 Peak | 中（需要醫師儀表板） |
| P1 | MOT-6 首次兌換 | 積點有價值的具體證明 | 中（需要活水夥伴合作） |
| P2 | MOT-5 社群引爆 | 從眾者的傳播引擎 | 中高（排行榜 + 群組功能） |
| P2 | MOT-8 故事分享 | 探索者的身分鎖定 | 低（內容生成 + 分享連結） |

## 七、結論：SYNC 的破壞式創新定位

```
    Clayton Christensen 的破壞式創新框架：

    性能
     │
     │     ┌─ 既有解決方案的性能軌跡

     │╲
     │ ╲    ← 90% 的人在這裡被忽視
     │  ╲
     │   ╲   ┌─ SYNC 的性能軌跡
     │    ╲ ╱   （從 LINE Bot 開始，逐步上升）
     │     ╳
     │    ╱ ╲
     │   ╱   ╲─ 傳統方案繼續提升
```

> **Project SYNC 的增長引擎不是功能，是情感。不是 App，是「被看見」。不是代幣，是「爸爸今天有走路」那一刻的安心。**
