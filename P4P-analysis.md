# 台灣 P4P（論質計酬）制度分析與 Project SYNC 策略對接

## Pay for Performance × AI-Driven Behavioral Prescription

---

> **摘要**：本報告系統梳理台灣全民健保 P4P（Pay for Performance，論質計酬）制度的發展脈絡、運作機制、實證成效與政策演進方向，並分析 Project SYNC 如何在此制度基礎上建立「診間外行為落實」的價值定位，成為 P4P 制度的延伸執行層。

## 一、P4P 制度概述

### 1.1 什麼是 P4P？

P4P（Pay for Performance，論質計酬）是一種**以醫療品質成果為給付依據**的支付制度，取代傳統「論量計酬」（Fee-for-Service）中以診療次數決定收入的模式。核心理念是：

- **醫療機構的收入與病患健康結果掛鉤**
- 鼓勵預防性照護、慢性病管理與持續追蹤
- 透過品質指標（Quality Indicators）衡量照護成效

### 1.2 台灣 P4P 發展史

| 年份 | 里程碑 |
|------|--------|
| **2001** | 健保局啟動「糖尿病論質計酬試辦計畫」，為台灣首個 P4P 方案 |
| **2006** | 擴展至氣喘、乳癌、子宮頸癌等疾病領域 |
| **2010** | 約 30% 糖尿病患者納入 P4P 計畫 |
| **2011** | 新增「初期慢性腎臟病（early-CKD）P4P」方案 |
| **2015** | 糖尿病 P4P 覆蓋率提升至 43% |
| **2017** | 整合為「家庭醫師整合性照護計畫」體系 |
| **2021** | 約 46% 第 2 型糖尿病患者參與，A1c < 8% 達標率 81.4% |
| **2022** | 整併「糖尿病及初期慢性腎臟病照護整合方案」，至 2023 年納入 18 萬人 |
| **2024** | 升級為「家醫計畫 2.0」+ 推出「三高防治 888 計畫」|
| **2025** | 計畫推出「家醫大平台」，整合 AI 風險分級模型 |

### 1.3 涵蓋範圍

目前台灣 P4P 計畫涵蓋的主要疾病領域：

| 疾病 | 啟動年份 | 覆蓋規模 |
|------|---------|---------|
| 第 2 型糖尿病 | 2001 | ~46% 糖尿病患者（約 100 萬人） |
| 初期慢性腎臟病（CKD） | 2011 | 18 萬人（2023 年） |
| 氣喘 | 2006 | — |
| 乳癌 | 2006 | — |
| COPD（慢性阻塞性肺病） | 後續加入 | — |
| 結核病 | 後續加入 | — |

### 2.2 給付結構

以糖尿病 P4P 為例：

| 給付項目 | 金額（USD） | 說明 |
|---------|-----------|------|
| 收案費 | $13/案 | 首次收案時支付 |
| 追蹤門診費 | $21/次 × 3 次 | 每季至少回診一次 |
| 年度評估費 | $27/年 | 完成年度檢查 |
| 醫師診察費 | $60/年 | 照護完整性獎勵 |
| **年度合計** | **~$108/患者** | 完成完整照護週期 |

**品質指標與獎勵機制**：
- 糖化血色素（HbA1c）< 8% 達標率
- 眼底檢查完成率
- 腎功能追蹤率
- 血脂控制率
- 足部檢查完成率

達標的團隊可獲得額外的品質獎勵金與盈餘回饋。

## 三、P4P 實證成效

### 3.1 臨床結果

#### 糖尿病 P4P

| 指標 | P4P 組 | 非 P4P 組 | 效果 | 資料來源 |
|------|--------|----------|------|---------|
| 存活率 | 顯著較高 | 基準 | HR = 0.76 (95% CI: 0.64-0.92, p=0.004) | [PubMed: 27399144](https://pubmed.ncbi.nlm.nih.gov/27399144/) |
| HbA1c < 8% 達標率 | 81.4% | — | 2021 年數據 | 國衛院電子報 |
| 追蹤完成率 | 72.5% | — | 2021 年數據 | 國衛院電子報 |
| 心血管事件 | 顯著降低 | 基準 | 多篇研究一致 | [PubMed: ScienceDirect](https://www.sciencedirect.com/science/article/pii/S009174352100356X) |
| 低血糖急診 | 顯著減少 | 基準 | 急診就醫次數下降 | [Oxford Academic](https://academic.oup.com/heapol/article/29/6/732/574534) |
| 下肢截肢風險 | 顯著降低 | 基準 | 全國世代研究 | ScienceDirect |

#### 慢性腎臟病 P4P

| 指標 | P4P 組 | 非 P4P 組 | 效果 | 資料來源 |
|------|--------|----------|------|---------|
| 疾病惡化率 | 0.44/100人月 | 0.69/100人月 | CS-HR = 0.61 (95% CI: 0.58-0.64, p<.0001) | [IJHPM](https://www.ijhpm.com/article_4030.html) |
| 糖尿病 + CKD 雙重參與 | — | — | CS-HR = 0.60 (95% CI: 0.54-0.67) | 同上 |

**關鍵發現**：同時參與糖尿病 P4P 與 CKD P4P 的患者，疾病惡化風險降低 40%，呈現**疊加效應**。

### 3.2 成本效益

根據 2022 年發表的成本效益分析研究：

- P4P 計畫具有**正向淨值**（positive net value）
- 降低住院、急診等高成本醫療利用
- 長期可延長壽命並降低失能
- 每投入 $1 的 P4P 獎勵金，估計節省 $2-5 的下游醫療費用（依疾病嚴重度而異）

> 參考：[Assessing Taiwan's P4P program: a cost-benefit net value approach](https://pubmed.ncbi.nlm.nih.gov/35995886/)

### 3.3 現行挑戰

根據 PMC 回顧文獻，台灣 P4P 計畫面臨的主要挑戰：

| 挑戰 | 具體問題 | 影響程度 |
|------|---------|---------|
| **基層參與率低** | 僅 8% 的 11,000 家獨立診所參與 P4P | ★★★★★ |
| **獎勵誘因不足** | 慢性照護活動的給付偏低 | ★★★★ |
| **IT 基礎設施落後** | 數位照護模式支撐不足 | ★★★★ |
| **以醫院為中心** | 基層靈活性受限 | ★★★ |
| **覆蓋率上限** | 約 50% 糖尿病患者未被覆蓋 | ★★★★ |
| **行為介入空白** | 診間外的行為落實缺乏追蹤 | ★★★★★ |

> 參考：[Chronic care model in the diabetes P4P program in Taiwan](https://pubmed.ncbi.nlm.nih.gov/33995846/)

## 四、P4P 政策演進方向（2024-2027）

### 4.1 三高防治 888 計畫

衛福部於 2024 年推出的旗艦慢性病計畫：

**三個 80% 目標**：
1. **80%** 三高患者納入照護網（目前 56%，即 301 萬人）
2. **80%** 接受生活型態諮詢
3. **80%** 三高控制達標率

**經費規模**：
- 534 萬名三高患者
- 三高相關疾病年花費近 **1,700 億元**（占健保支出 1/5）
- 目標涵蓋 **400 萬人以上**

### 4.2 家醫大平台（2025 年底推出）

健保署投入 **4.5 億元** 建構：

| 功能 | 說明 |
|------|------|
| 個人健康資料整合 | 整合健保、健檢及數位工具數據 |
| AI 風險分級模型 | 心血管疾病與糖尿病優先 |
| 分流照護 | 依風險等級分配照護資源 |
| 跨機構協作 | 醫院、診所、藥局、長照、居家服務串聯 |

### 4.3 數位照護試辦

健保署選定 **4 家數位照護廠商** 試辦：
- 智抗糖
- WaCare
- 智齡科技
- 遠傳電信

核心模式：**達成具體健康改善則有額外獎勵**（結果導向數位介入）

### 4.4 基層診所數位化（2025-2027）

- 目標：**80% 基層診所導入雲端 HIS**
- 整合預約、電子處方箋等功能
- 政府補助推動

## 五、P4P 與 Project SYNC 的策略對接

### 5.1 結構性機會：P4P 的「最後一哩路」問題

```
P4P 制度覆蓋範圍：

  ┌─ 診間內 ──────────────────────────────────────┐
  │                                               │
  │  ✅ 醫療團隊組成（醫師、衛教師、營養師）          │
  │  ✅ 檢驗追蹤（HbA1c、eGFR、血脂）               │
  │  ✅ 品質指標監測（年度評估）                      │
  │  ✅ 給付獎勵結構                                │
  │                                               │
  └───────────────────────────────────────────────┘

  ┌─ 診間外 ──────────────────────────────────────┐
  │                                               │
  │  ❌ 日常行為追蹤（運動、飲食、服藥、睡眠）        │
  │  ❌ 即時行為回饋與激勵                           │
  │  ❌ 社區活動參與紀錄                             │
  │  ❌ 行為數據回傳醫療端                           │
  │  ❌ 病友互動與社會支持量化                        │
  │                                               │
  │      ← Project SYNC 介入區間 →                  │
  │                                               │
  └───────────────────────────────────────────────┘
```

**P4P 制度最大的結構性弱點**：醫療團隊只能在病人每 3 個月回診一次時確認結果，但**中間 90 天的行為落實完全是黑箱**。

Project SYNC 的定位：**填補 P4P 的「診間外 90 天」行為追蹤空白**。

### 5.2 價值主張對接

| P4P 現有問題 | Project SYNC 解決方案 | 對接機制 |
|-------------|---------------------|---------|
| 基層診所參與率僅 8% | 降低參與門檻：LINE Bot 無需安裝 App | 醫師只需在 LINE 上「轉介」病人 |
| 診間外行為無法追蹤 | AI 代理人每日行為提醒與紀錄 | hEXP 積點量化行為完成度 |
| 衛教效果難以衡量 | Gemini AI 個人化健康問答 | 互動紀錄作為衛教成效指標 |
| 病友支持團體 450+ 但鬆散 | 遊戲化社區任務串聯線上線下 | 社會處方箋 × 社區據點打卡 |
| 缺乏行為誘因設計 | hEXP 積點可兌換實質獎勵 | 活水夥伴提供兌換場景 |
| 數位照護基礎設施不足 | 雲端平台 + LINE 介面 + API 串接 | 可對接家醫大平台 |

### 5.3 P4P 品質指標 × SYNC 行為指標映射

Project SYNC 可將日常行為數據轉化為 P4P 品質指標的**預測性指標**：

| P4P 品質指標 | SYNC 行為指標 | 預測邏輯 |
|-------------|-------------|---------|
| HbA1c < 8% | 每日飲食紀錄完成率、運動次數 | 行為完成度高 → HbA1c 達標率提升 |
| 回診追蹤率 | LINE 推播確認 + 提醒已讀率 | 主動提醒 → 回診到診率提升 |
| 眼底/足部檢查率 | 年度任務推播 + 完成打卡 | 任務提醒 → 檢查完成率提升 |
| 腎功能追蹤率 | 飲水量紀錄、鈉攝取追蹤 | 行為改善 → eGFR 穩定 |
| 生活型態諮詢率（888 新增） | AI 對話次數 + 主題覆蓋率 | 數位諮詢 → 達成 80% 目標 |

### 5.4 商業模式對接

#### 短期（Phase 0-1）：作為 P4P 團隊的「行為追蹤延伸工具」

```

├── 醫療機構 SaaS 費用（每月 $X/醫師）
├── P4P 品質獎勵金分潤（達標協作費）
└── 數位照護試辦補助申請

價值論述：
  SYNC 幫助您的 P4P 團隊：
  ✅ 提升回診追蹤率（從 72.5% → 85%+）
  ✅ 提升 HbA1c 達標率（從 81.4% → 90%+）
  ✅ 獲得更多 P4P 品質獎勵金
  ✅ 減少團隊人工追蹤的時間成本
```

#### 中期（Phase 2-3）：作為「家醫大平台」的行為數據供應商

```
收入來源：
├── 家醫大平台 API 串接費用
├── 行為數據授權（匿名聚合）
├── 活水夥伴的場域行銷分潤
└── 888 計畫生活型態諮詢計點

定位升級：
  從「診所工具」→「生態系平台」
  從「追蹤行為」→「創造行為」
  從「P4P 輔助」→「P4P + 社會處方箋 + 行為經濟」三位一體
```

## 六、關鍵建議：如何讓 SYNC 成為 P4P 制度的「行為引擎」

### 6.1 Phase 0 — 立即行動（Google.org 提案前）

1. **P4P 對接論述**：在 Google.org 提案中明確寫入 SYNC 與 P4P 制度的對接機制
2. **取得醫療端 LOI**：找到 1-2 位已參與 P4P 的家醫科醫師簽署意向書
3. **量化潛在影響**：使用本報告數據，計算 SYNC 可能帶來的 HbA1c 達標率提升
4. **數位照護試辦**：評估能否以第 5 家身份申請 888 計畫的數位照護試辦

### 6.2 Phase 1 — MVP 開發聚焦

1. **優先開發 P4P 追蹤功能**：
   - LINE Bot 回診提醒（對應追蹤率指標）
   - 每日行為打卡（飲食、運動、服藥）
   - AI 衛教問答（對應生活型態諮詢率）

2. **設計「醫師儀表板」**：
   - 顯示 P4P 品質指標進度
   - 呈現病人行為數據摘要
   - 標記需要介入的高風險病人

3. **建立成效衡量框架**：
   - P4P 品質指標前後比較（介入前 vs 介入後）
   - 行為完成率與臨床指標的相關性分析
   - 為未來的研究論文奠定數據基礎

### 6.3 Phase 2 — 制度化對接

1. **發表研究論文**：SYNC 對 P4P 品質指標的影響（前後比較研究）
2. **申請家醫大平台 API 串接**：成為官方認可的行為數據源
3. **推動「行為處方箋」正式化**：從非正式的行為建議 → 可被 P4P 計點的正式處方
4. **串接 888 計畫生活型態諮詢**：AI 對話紀錄作為諮詢達標的佐證

## 七、風險與限制

| 風險 | 說明 | 緩解策略 |
|------|------|---------|
| P4P 政策變動 | 健保署可能調整 P4P 給付結構 | 保持多元收入源，不完全依賴 P4P |
| 醫師端採用阻力 | 醫師已有繁重的 P4P 文書工作 | 設計為「減輕負擔」而非「增加工作」 |
| 數據隱私 | 行為數據回傳醫療端涉及個資法 | TW DIW VC 化的選擇性揭露機制 |
| 既有競爭者 | 智抗糖等已在試辦名單中 | 差異化定位：行為經濟 + 社區連結 |
| 規模化門檻 | P4P 以醫院為中心，基層診所參與低 | 從友善診所切入，逐步擴展 |

---

## 八、結論

台灣 P4P 制度經過 25 年的發展，已建立完善的品質指標體系、給付結構和共照網絡。然而，其核心弱點——**診間外 90 天的行為追蹤空白**——至今未被有效解決。

Project SYNC 的價值定位恰好對準這個結構性缺口：

1. **P4P 需要行為數據** → SYNC 提供每日行為追蹤
2. **P4P 需要病人依從** → SYNC 提供遊戲化激勵
3. **P4P 需要數位基建** → SYNC 以 LINE Bot + AI 代理人填補
4. **888 計畫需要生活型態諮詢** → SYNC 的 AI 對話就是數位版本的諮詢
5. **家醫大平台需要數據源** → SYNC 可成為行為數據 API 供應商

隨著 888 計畫、家醫大平台、數位照護試辦等政策在 2025-2027 年間陸續落地，**Project SYNC 正處於一個罕見的制度窗口期**——P4P 制度已成熟到需要「行為執行層」，而這正是 SYNC 的核心能力。
