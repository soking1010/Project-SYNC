# Project SYNC — 產品路線圖

## AI 驅動全人健康守護計畫：從行為處方到健康經濟生態系

---

> ### 宣言
>
> **運用自己的行為數據，滋養數位孿生的靈魂。**
>
> 建構一個區塊鏈底層的點數交換機制，讓使用者在參與遊戲化行為任務過程，
> 連結實體場域、行為，變成可以被授權的行銷 Token。

---

## 產品願景

每一個慢性病患者的日常行為——走路、社交、飲食、服藥、睡眠——都是有價值的數據。這些數據不應該只是被動地躺在醫療系統裡等待被讀取，而應該成為驅動自身健康的燃料。

Project SYNC 要做的，是讓病人從「被管理的對象」變成「自主經營健康的行動者」。透過遊戲化行為任務，病人的每一步行動都在滋養自己的數位孿生——一個忠實映射其生理狀態、行為模式與社會連結的數位分身。而這些行為數據，經由區塊鏈底層的授權機制，將轉化為可流通、可驗證、可增值的健康 Token，在醫療、社區、商業三方生態系中自由流動。

但一個健康的經濟生態系，不能只靠「供給」與「需求」自然配對。**它需要活水——持續注入新鮮內容、豐富場景與多元獎勵的力量。** 我們稱這股力量為「活水夥伴」。

### 市場規模估算

```
TAM（Total Addressable Market）：
  台灣慢性病患者 × 健康管理市場
  ≈ 1,000 萬慢性病人口 × 年均自費健康支出 NT$12,000
  ≈ NT$1,200 億（~US$3.8B）

SAM（Serviceable Available Market）：
  糖尿病 + 高血壓 × 數位健康管理意願人口
  ≈ 500 萬人 × 20% 數位化意願 × NT$6,000/年
  ≈ NT$60 億（~US$190M）

SOM（Serviceable Obtainable Market）— Phase 1-2：
  合作診間的病患數 × 行為經濟產值
  ≈ 5,000 人 × NT$3,000/年（行為積點經濟 + 數據授權）
  ≈ NT$1,500 萬（~US$475K）
```

### 競爭格局

| 競爭者類型 | 代表 | 與 SYNC 差異 |
|-----------|------|-------------|

**SYNC 的獨特定位：「診間外行為落實」的最後一哩路** — 目前沒有任何產品明確佔位於此。

## 活水夥伴：生態系的流動性引擎

### 什麼是活水夥伴？

在加密貨幣的世界中，造市商（Market Maker）扮演關鍵角色：他們同時在買方與賣方掛單，為市場提供流動性，確保交易不會因缺乏對手方而停滯。沒有造市商，市場就是死水。

**活水夥伴，就是 Project SYNC 生態系中的流動性引擎。**

他們不是被動等待病人上門的服務提供者，而是主動注入「行為流動性」的創造者——設計有吸引力的任務、創造有意義的場景、投入可兌換的獎勵，讓整個健康經濟體保持活絡與新鮮。

### 活水夥伴的三重角色

| 角色面向 | 類比造市商功能 | 在 SYNC 中的實踐 |
|---------|--------------|----------------|
| **任務供給者** | 掛出賣單（提供流動性） | 設計並發布行為任務，讓使用者有事可做、有目標可追 |
| **獎勵注入者** | 掛出買單（吸收流動性） | 投入實體或虛擬獎勵，讓積點有兌換去處 |
| **場景創造者** | 穩定市場價差（降低摩擦） | 打造線上線下的互動場景，降低使用者參與門檻 |

### 誰可以成為活水夥伴？

**我們歡迎所有願意為健康生態系注入活水的組織與個人加入。**

| 類型 | 他們帶來什麼 | 他們獲得什麼 |
|------|------------|------------|
| **創意工作者** — 插畫家、設計師、內容創作者 | 設計任務視覺、創造數位收藏品、製作衛教內容 | 積點分潤、作品曝光、行為數據洞察（匿名聚合） |
| **活動舉辦者** — 運動教練、藝術治療師、社區導覽員 | 開設實體活動任務、提供場域體驗、帶領團體互動 | 積點分潤、參與者數據、醫療機構轉介導流 |
| **非營利組織** — 病友協會、銀髮服務團體、社區發展協會 | 串接弱勢族群、提供社區網路、倡議健康議題 | 積點補助金、影響力報告、P4P 社會處方箋成效數據 |
| **在地商家** — 餐廳、健身房、藥局、農場 | 提供兌換點、設計場域限定任務、贊助獎勵 | 精準導流、行為偏好數據授權、品牌健康形象 |

### 活水夥伴的經濟循環

```
活水夥伴 ──── 投入任務 + 獎勵 ────► 生態系池子
    ▲                                    │
    │                                    ▼
    │                              使用者完成任務
    │                              產生 hEXP / 兌換積點
    │                                    │
    │                                    ▼
    └──── 獲得行為洞察 + 導流 ◄──── 使用者兌換積點
          + 分潤
```

**關鍵機制：行為價差收益（Behavioral Spread）**

如同造市商從買賣價差中獲利，活水夥伴從「行為價差」中獲取價值——他們投入的任務與獎勵成本，換回的是經過行為驗證的精準受眾數據與導流效果。這個價差越健康（使用者體驗越好、參與度越高），活水夥伴的回報越高。這不是零和博弈，而是正和循環。

## Phase 1：MVP — 行為處方箋引擎（2026 Q2-Q3）

### 目標

在合作診間啟動第一批慢性病患者的行為處方箋試行，**驗證「診間外行為落實」的可行性，累積投資人會問的 traction 數據。**

### Phase 1 的驗證假設

> 募資評估洞察：Phase 1 結束時必須能回答投資人的五個問題：
> 1. 月活用戶有多少？
> 2. 病人的留存率是多少？
> 3. 行為處方箋完成率是否優於對照組？
> 4. 醫師願意持續使用嗎？
> 5. 有多少人主動參與社區活動？

### 核心功能

#### 1.1 行為處方箋系統

醫師於診間開立「行為處方箋」，AI 代理人根據病人的即時生理數據（穿戴裝置）與生活情境，主動觸發遊戲化微任務。

**任務生命週期引擎：**

```
任務 → 觸發 → 過程 → 完成條件 → 發放
```

四種任務來源：

| 來源 | 說明 | 範例 |
|------|------|------|
| 醫師處方任務 | 由醫師根據病情開立 | 「每日步行 3000 步，持續 14 天」 |
| 社區據點任務 | 由銀髮據點、藝術治療團體發布 | 「參加本週三的太極拳班」 |
| 病友互助任務 | 由活躍參與者自主創建 | 「揪團週末公園走路挑戰」 |
| 活水夥伴任務 | 由活水夥伴設計並投放 | 「完成健康食譜挑戰，兌換農場鮮蔬箱」 |

#### 1.2 簡化版行為積點系統

> 募資評估洞察：Phase 1 大幅簡化 Token 經濟。只保留 hEXP 作為參與度指標 + 簡單的兌換積點（不叫 hMile、不上鏈）。完整的雙幣經濟留到 Phase 3，並需要專業 Token 經濟顧問做模型壓力測試。

**Phase 1 積點設計（刻意簡化）：**

| 機制 | 說明 |
|------|------|
| **hEXP（健康經驗值）** | 不可交易，反映投入程度。完成任務即獲得，連續參與有加成。 |
| **兌換積點** | 平台內點數（非 Token），可兌換合作商家的折扣或服務。由平台 + 少數種子活水夥伴注入。 |

**為什麼不在 Phase 1 就上完整雙幣系統？**

- 高齡使用者需要最簡單的理解路徑：「做任務 → 得積分 → 換東西」
- 沒有足夠使用者規模時，複雜的 Token 經濟機制沒有意義
- 法規合規需要充分準備時間，hMile 鏈上化的法務風險不應在 Phase 1 承擔
- 投資人更想看到「行為改變是否發生」而非「Token 系統多精巧」

**Phase 1 積點參數（初始值）：**

| 行為事件 | hEXP | 兌換積點 | 驗證方式 |
|---------|------|---------|---------|
| 每日步行達標（3000 步） | 10 | — | 穿戴裝置 API |
| 社區運動課出席 | 20 | 2 | QR/NFC 簽到 |
| 參加社區活動 | 15 | 1 | QR/NFC 簽到 |
| 完成病友互助任務 | 25 | 3 | 雙方確認 |
| 回診完成 | 30 | 5 | 健保系統對接 |
| 行為處方箋階段完成 | 50 | 10 | AI 綜合評估 |
| 每日飲食紀錄 | 5 | — | 自主回報 |

連續天數加成與衰減機制同代幣經濟白皮書設計，但簡化呈現——使用者只看到「連續 N 天，積分加倍！」的提示，不需理解背後公式。

#### 1.3 漸進式參與路徑（簡化版）

> 募資評估洞察：7 級參與路徑對高齡使用者過於複雜。Phase 1 簡化為 4 級，Phase 2 再擴展。

**Phase 1 參與層級（4 級）：**

| 層級 | 角色 | 觸發點 | 解鎖功能 |
|------|------|--------|---------|
| L0 | 訪客 | 在社區據點掃 QR Code 或收到 LINE 推薦 | 瀏覽資訊、領取體驗包 |
| L1 | 參與者 | 透過 LINE 登入 | 接收微任務推播、累積 hEXP、兌換積點 |
| L2 | 健康玩家 | 完成首張行為處方箋 | 查看健康儀表板、社交功能、邀請好友 |
| L3 | 夥伴 | 社區據點 / 活水夥伴 / 醫療機構 | 任務管理、行為數據 Dashboard |

**完整七級路徑（含 L4-VP 活水夥伴認證等）保留於代幣經濟白皮書，在 Phase 2-3 逐步啟用。**

## Phase 2：社會處方箋 × 去中心化信任（2026 Q3-Q4）

### 目標

導入 TW DIW 可驗證憑證，實現跨機構的去中心化資料授權與社會處方箋自動媒合。**同步啟動雙幣經濟模型（hEXP + hMile），為 Phase 3 鏈上化做準備。**

### 核心功能

#### 2.1 行為處方箋 VC 化

將行為處方箋設計為 W3C Verifiable Credential 格式：

| 角色 | TW DIW 身分 | 職責 |
|------|------------|------|
| 醫師／醫療機構 | Issuer | 簽發行為處方箋 VC |
| 病人 | Holder | 持有並自主授權出示 |
| 社區據點 | Verifier | 驗證資格，無需存取完整病歷 |
| AI 代理人 | Mediator | 根據 VC 內容媒合最適社區資源 |
| 活水夥伴 | Content Provider | 根據 VC 標籤設計對應的任務與獎勵 |

**選擇性揭露實踐：** 病人到銀髮據點時，僅出示「活動類別」與「有效期間」，據點無需知道完整病歷即可確認資格。

#### 2.2 社會參與 VC 紀錄

社區據點簽發的參與 VC，回傳醫師後台時具備密碼學可驗證性，可作為健保論質計酬（P4P）的客觀依據。

#### 2.3 hMile 正式啟用（中心化版本）

Phase 2 正式導入完整雙幣模型（hEXP + hMile），但 hMile 仍為中心化平台積點，尚不上鏈：

| 代幣 | 性質 | 產生方式 | 消耗方式 |
|------|------|---------|---------|
| **hEXP**（健康經驗值） | 不可交易，代表投入程度 | 執行行為處方、完成微任務、出席活動 | 解鎖進階功能、提升數位孿生等級 |
| **hMile**（健康里程幣） | 可授權流通的行銷積點 | 累積 hEXP 達標後轉換、活水夥伴注入 | 兌換健檢折扣、社區服務、合作商家優惠 |

此階段需完成：
- 聘請專業 Token 經濟顧問，對雙幣模型進行壓力測試
- 法規顧問評估 hMile 鏈上化的合規路徑
- 活水夥伴分潤機制的小規模試行

#### 2.4 任務編輯器

賦能社區據點營運者與活水夥伴自主建立活動任務：

```
選擇活動類型 → 基本設定 → 參與前提條件 + 觸發對象
     → 設定訊息腳本 → 設定完成條件 → 設定獎勵物
```

**活水夥伴專屬功能：**

- 任務模板庫（預設健康主題模板，降低設計門檻）
- 獎勵投放排程（設定積點獎勵的發放時機與額度）
- A/B 測試工具（比較不同任務設計的參與率）
- 行為洞察報表（匿名聚合的參與者行為模式分析）

#### 2.5 醫師端行為 Dashboard

將病人的社會參與度、行為處方執行率、社區活動偏好等數據視覺化呈現，作為回診時的診間參考依據。

## Phase 3：健康 Token 經濟 × 活水夥伴網路（2027 Q1-Q2）

### 目標

建構區塊鏈底層的點數交換機制，讓健康行為數據成為可授權的行銷 Token，並正式啟動活水夥伴開放招募，開啟商業可持續模式。

### 前提條件（Phase 3 Gate）

> 募資評估洞察：Token 鏈上化必須滿足以下前提才可啟動，否則風險過高。

| 條件 | 驗證指標 |
|------|---------|
| 使用者規模 | 月活 ≥ 1,000 人 |
| 生態活躍度 | 活水夥伴月活躍 ≥ 10 組 |
| 法規合規 | 取得法務意見書確認 hMile 不觸發虛擬通貨法規 |
| Token 經濟壓力測試 | 專業顧問完成模型驗證 |
| 團隊就位 | CTO + 法規顧問正式到位 |

### 核心功能

#### 3.1 hMile Token 鏈上化

將 hMile 從中心化點數升級為鏈上 Token，實現：

- **可追溯性：** 每筆 hMile 的產生都綁定一個可驗證的健康行為紀錄
- **可授權性：** 使用者自主決定將行為偏好授權給哪些商家，授權範圍透明可控
- **可審計性：** 健保署可獨立驗證 P4P 相關的行為紀錄，無需信任單一平台
- **流動性保障：** 活水夥伴的持續投入確保 hMile 生態系的供需平衡

#### 3.2 行為數據授權市集

使用者的行為偏好（非醫療隱私）經匿名化與聚合後，可授權給合作商家用於精準行銷：

| 資料類型 | 授權範例 | 回饋形式 |
|---------|---------|---------|
| 運動偏好 | 運動品牌推薦 | hMile + 商品折扣 |
| 社區活動參與 | 在地商家導流 | hMile + 餐飲優惠 |
| 飲食習慣 | 健康食品推薦 | hMile + 試用品 |
| 步行路線 | 場域行銷觸發 | hMile + 據點限定獎勵 |
| 任務偏好 | 活水夥伴精準投放 | hMile + 專屬體驗 |

**關鍵原則：** 這不是「賣數據」，而是使用者主動授權自己的行為軌跡給特定場域，換取實質回饋。每一次授權都是一筆鏈上紀錄，使用者隨時可撤銷。

#### 3.3 活水夥伴開放平台

正式上線活水夥伴自助入駐系統：

| 入駐流程 | 說明 |
|---------|------|
| 身分認證 | 組織或個人透過 TW DIW 完成身分驗證 |
| 角色選擇 | 選擇定位：創意工作者 / 活動舉辦者 / 非營利組織 / 在地商家 |
| 資源配置 | 設定可投入的獎勵額度、任務產出頻率 |
| 試營運 | 為期 30 天的試營運期，系統評估內容品質與參與成效 |
| 正式上線 | 通過審核後獲得完整活水夥伴工具權限 |

**活水夥伴分潤機制：**

```
使用者完成活水夥伴設計的任務
    → 系統紀錄任務歸屬
    → 使用者獲得 hEXP + hMile
    → 活水夥伴獲得：
        ├── hMile 分潤（依任務完成數計算）
        ├── 行為洞察報表（匿名聚合）
        └── 生態系信譽積分（影響曝光排序）
```

#### 3.4 數位孿生進化

累積的行為數據持續滋養使用者的數位孿生模型：

- **生理面：** 穿戴裝置數據、健康存摺病歷
- **行為面：** 任務完成紀錄、運動與飲食模式
- **社會面：** 社區參與度、社交連結強度
- **心理面：** AI 對話紀錄推估的心理韌性指標

數位孿生成為使用者理解自身狀態的鏡子，也成為 AI 代理人個人化介入的依據。

#### 3.5 場域 IoT 觸發網路

社區據點佈建 QR Code / NFC 觸發點，病人到場即觸發：

```
病人到達社區據點
    → 掃碼/感應簽到
    → 觸發社會處方箋參與紀錄（VC）
    → 啟動當日微任務推播（含活水夥伴投放任務）
    → 完成後自動發放 hEXP + hMile
    → 據點端 Dashboard 即時更新
    → 活水夥伴端數據同步
```

## Phase 4：生態系規模化（2027 Q3+）

### 目標

從單一試辦診間擴展至多元醫療體系與縣市級社區網路，將活水夥伴網路發展為自運轉的開放生態系。

### 關鍵方向

**橫向擴展——疾病別：** 從糖尿病 / 高血壓擴展至其他慢性病（COPD、憂鬱症、失智前期），每個疾病別設計專屬的行為處方箋模板與微任務庫。

**縱向擴展——場域：** 從醫療體系擴展至長照機構、職場健康促進、學校衛教，每個場域有獨立的任務編輯器與 Dashboard。

**生態擴展——活水網路：** 開放 API 讓更多活水夥伴加入，形成「活水夥伴注入 → 使用者行為 → Token 流通 → 數據回饋 → 更精準的注入」的自增強飛輪。

**社會影響力擴展——開放邀請：**

> **我們歡迎每一位願意為社區健康注入活水的夥伴。**
>
> 無論你是一位用畫筆傳遞溫暖的插畫家、一位帶領長者做椅子瑜珈的教練、一個為罕病家庭奔走的協會、還是一家願意提供健康餐點的社區小店——你都可以成為活水夥伴。
>
> 你的創意就是一帖處方，你的活動就是一劑疫苗，你的關懷就是一份社會處方箋。
>
> **加入 Project SYNC，讓健康不只是醫療系統的事。**
